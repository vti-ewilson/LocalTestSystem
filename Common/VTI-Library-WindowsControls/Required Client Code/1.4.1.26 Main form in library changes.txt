Main form migration procedure

1. Pull latest library version

2. Change all MainForm type references to VTIWindowsControlLibrary.Forms.MainForms.MainForm

	2a. In Program.cs change the mainForm field declaration from 
		"private static Forms.MainForm mainForm;" to 
		"private static VTIWindowsControlLibrary.Forms.MainForms.MainForm mainForm;"

	2b. In Program.cs change the MainForm property declaration from 
		"public static Forms.MainForm MainForm { get { return mainForm; } }" to 
		"public static  VTIWindowsControlLibrary.Forms.MainForms.MainForm MainForm { get { return mainForm; } }"

	2c. In Program.cs change the line 
		"mainForm = new Forms.MainForm();" to "mainForm = new VTIWindowsControlLibrary.Forms.MainForms.MainForm();"

	2d. In Machine.cs change the _MainForm field declaration from 
		"protected MainForm _MainForm;" to 
		"protected VTIWindowsControlLibrary.Forms.MainForms.MainForm _MainForm;"

	2e. In Machine.cs change the MainForm property declaration from 
		"public static MainForm MainForm { get { return Instance._MainForm; } }" to 
		"public static VTIWindowsControlLibrary.Forms.MainForms.MainForm MainForm { get { return Instance._MainForm; } }"

3. Add these functions to bottom of the Machine class in Machine.cs

// Handle any special login procedures here
public static void OnStartup() {
	ManualCommands.Login();
}

public static void OnShutdown() {
        ManualCommands.ShutDown(); // May be named differently but call whatever shutdown command exists
}

public static void ConstantTick() {

}

4. Copy paste timerSlidePanels_tick code from existing MainForm into ConstantTick()
	- If there are references to Mainform members in this code those will need to be either moved to Machine or accessed through the Machine.Mainform object
		- ex: replace "this." with "MainForm."

Troubleshooting:

- If there are missing references to iText
	- try right clicking on packages.config in VTIWindowsControlLibrary and selecting "Migrate packages.config to PackageReference"
